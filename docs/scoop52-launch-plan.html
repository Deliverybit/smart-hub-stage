7. Critical Bottlenecks & Fixes

IssueImpactFixPriority
Headline enrichment not cachedUp to 10 AV news calls per user per screener pageCache enrich_headline_sentiment (15 min TTL)Must-have
Per-replica Streamlit cacheCold-start API spikes on deploy/scaleRolling deploys; Phase 2 Redis workerShould-have
Community Cloud for prodSleep, 1 GB RAM, no SLADocker on Render/RailwayMust-have
Search page abuseUnbounded per-ticker AV callsCloudflare rate limit + Redis cacheShould-have
Alpha Vantage usage at 1200 rpm: Scheduled precompute ≈ 110–140 rpm sustained (~10% of capacity). Uncached headline enrichment is the remaining user-scaled risk — fix before launch.

8. Pre-Deploy Checklist

Must-Have Before Launch

  1. Dockerfile + .dockerignore
  2. Cache headline enrichment (15-min TTL)
  3. Create smart-hub-prod Supabase project
  4. Prod secrets on host (APP_ENV, DATABASE_URL, ALPHA_VANTAGE_API_KEY)
  5. Run migrations: python admin_tools/run_migrations.py
  6. Cloudflare + custom domain with WebSockets enabled
  7. Separate prod Alpha Vantage key from staging

Should-Have (First Month)

  1. Search page rate limit (Cloudflare or app-level)
  2. Rolling deploys (one replica at a time)
  3. Sentry + uptime monitoring
  4. Documented SCREENER_CACHE_VERSION bump process
