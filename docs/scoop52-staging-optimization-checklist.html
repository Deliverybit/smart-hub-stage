1. Strategy: Staging First, Prod Later

Do not create production yet unless you are within 1–2 weeks of public launch. Use staging to validate optimizations safely, then promote to prod when all gates pass.

3. Deploy prod host (Render/Railway) ← when ready to go live

7. Staging “Ready for Prod” Gates

Only create prod and go live when all are true:

  • All screeners load reliably after warm-up (no empty tables under normal AV usage)
  • Headline caching deployed — 2nd page load within 15 min is noticeably faster
  • Terms consent writes to smart-hub-stage every time
  • “Clear cache and refresh” recovers without manual intervention
  • Dockerfile runs locally without errors
  • No recurring Alpha Vantage rate-limit errors during normal browsing
